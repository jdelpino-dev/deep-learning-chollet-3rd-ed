# ------------------------------------------------------------------------------
# JAX Backend — Apple Silicon GPU
#
# IMPORTANT: jax, jaxlib, and jax-metal must be a compatible trio.
# Apple's jax-metal docs provide a compatibility table mapping
# jax-metal versions to supported jaxlib versions. Do NOT mix
# incompatible versions or JAX will silently fall back to CPU.
#
# jax-metal 0.1.1 (latest as of Feb 2026, released Oct 2024) requires
# jax >= 0.4.34 with jaxlib == jax. We pin 0.4.38 — within the supported
# range but close enough to the plugin's last-tested baseline. Newer JAX
# versions can work by setting ENABLE_PJRT_COMPATIBILITY=1 (per Apple's
# docs), but since the plugin hasn't been updated in 16+ months and is
# explicitly experimental, staying near the minimum reduces breakage risk.
#
# See: https://pypi.org/project/jax-metal/
# See: https://developer.apple.com/metal/jax/
# ------------------------------------------------------------------------------
jax==0.4.38
jaxlib==0.4.38
jax-metal==0.1.1


# ------------------------------------------------------------------------------
# TensorFlow — CPU only (no tensorflow-metal)
#
# TF is required even when training with JAX, because tf.data pipelines
# are used for data loading throughout chapters 8–17, and keras.datasets
# (MNIST, CIFAR, etc.) depends on it.
#
# GPU acceleration is NOT needed for TF in this book:
#   - All GPU-heavy training runs through JAX + jax-metal (primary backend)
#   - PyTorch GPU cells use MPS (built-in)
#   - TF is only used for: tf.data pipelines (CPU), keras.datasets (downloads),
#     and a handful of tf.GradientTape demos (ch. 3, 10) that run fine on CPU
#
# WHY NO tensorflow-metal?
#   As of Feb 2026, tensorflow-metal==1.2.0 (the only version with a
#   Python 3.12 wheel) has a broken rpath: its libmetal_plugin.dylib
#   expects _pywrap_tensorflow_internal.so at a Bazel-generated relative
#   path (_solib_darwin_arm64/...) that does not exist in TF 2.18 or 2.20's
#   installed package layout. tensorflow-metal==1.1.0 has no 3.12 wheel.
#   This is an Apple-side packaging bug with no workaround.
#   Dropping tensorflow-metal has zero functional impact on these notebooks.
# ------------------------------------------------------------------------------
tensorflow==2.20.0


# ------------------------------------------------------------------------------
